这一段代码解决的是 **“字母异位词分组” (Group Anagrams)** 问题。

**什么是字母异位词？**
比如 `"eat"`、`"tea"` 和 `"ate"`，它们包含的字母完全一样（都是 a, e, t），只是顺序不同。这段代码的目标就是把这些“成分相同”的单词分到一组里。

---

### 核心思路：寻找“通用的钥匙”

要分组，就得有个标准。这段代码最聪明的地方在于：**把单词按字母排序。**

* `"eat"` 排序后是 `"aet"`
* `"tea"` 排序后也是 `"aet"`
* `"ate"` 排序后还是 `"aet"`

这样，`"aet"` 就成了这一组单词共同的 **Key（钥匙/标签）**。

---

### 逐行代码拆解

#### 1. 定义容器

```java
Map<String, List<String>> map = new HashMap<String, List<String>>();

```

* **`Map`**: 想象成一个有很多抽屉的柜子。
* **`String` (Key)**: 抽屉上的“标签”（即排序后的字符串，如 `"aet"`）。
* **`List<String>` (Value)**: 抽屉里装的“名单”（即原始单词列表，如 `["eat", "tea"]`）。

#### 2. 遍历每一个单词

```java
for (String str : strs) {

```

这是一个“增强型 for 循环”，意思是：把字符串数组 `strs` 里的单词，一个接一个拿出来处理，当前拿到的叫 `str`。

#### 3. 制造“标签” (排序)

```java
char[] array = str.toCharArray(); // 把 "eat" 变成 ['e', 'a', 't']
Arrays.sort(array);               // 把数组排成 ['a', 'e', 't']
String key = new String(array);   // 把数组变回字符串 "aet"

```

这里是关键：不管原始单词长什么样，只要字母相同，算出来的 `key` 就一定相同。

#### 4. 存入抽屉 (核心逻辑)

```java
List<String> list = map.getOrDefault(key, new ArrayList<String>());

```

* **`getOrDefault(key, default)`**: 这是一个非常实用的方法。
* 它会问 Map：“柜子里有 `key`（比如 `aet`）这个抽屉吗？”
* **如果有**：把抽屉里的名单拿出来（赋值给 `list`）。
* **如果没有**：现场新建一个空的名单（`new ArrayList`）。



```java
list.add(str);     // 把当前的原始单词（比如 "eat"）塞进名单里
map.put(key, list); // 把更新后的名单放回抽屉里

```

#### 5. 返回结果

```java
return new ArrayList<List<String>>(map.values());

```

* **`map.values()`**: 把柜子里所有抽屉里的“名单”全部取出来。
* **`new ArrayList(...)`**: 把这些名单打包成一个大列表返回。

### 常用成员方法总结表

如果你对 Java 基础不太熟悉，看这张表就能明白代码里那些“点（`.`）”后面跟着的函数在干嘛：

| 方法 | 所属类 | 作用 (白话解释) |
| --- | --- | --- |
| **`toCharArray()`** | `String` | 把一串话拆成一个个字符，方便排序。 |
| **`Arrays.sort()`** | `Arrays` | 自动把数组按字母表顺序 A-Z 排好。 |
| **`getOrDefault()`** | `Map` | 有就拿出来，没有就给个默认的（防止报错）。 |
| **`put(k, v)`** | `Map` | 往柜子里存东西，指定标签 `k` 和内容 `v`。 |
| **`add()`** | `List` | 往列表（名单）最后加一个新名字。 |
| **`values()`** | `Map` | 只要柜子里的内容，不要外面的标签。 |

### 总结

这段代码就像是一个邮局分拣员：

1. 拿到一个信封（单词）。
2. 把信封上的字母重新排个序（做成标签）。
3. 看看有没有这个标签的格堆。
4. 有就扔进去，没有就新开一个格堆。
5. 最后把所有的格堆一起交上去。

这道题是 **“最长连续序列” (Longest Consecutive Sequence)**。

它的目标是在一堆乱七八糟的数字里，找到最长的一串能排成“1, 2, 3, 4...”这样连续的数字有多长。这段代码最精妙的地方在于：**它只从每一段连续数字的“头儿”开始数，不浪费时间。**

---

### 逐行注释分析

```java
class longestConsecutiveSolution {
    public int longestConsecutive(int[] nums) {
        // 1. 把所有数字丢进一个“去重盒子” (HashSet)
        // Set 的特点：1. 里面没有重复数字；2. 查某个数字在不在里面，速度极快（秒查）。
        Set<Integer> num_set = new HashSet<Integer>();
        for (int num : nums){
            num_set.add(num); // 把数组里的数一个一个存进去
        }

        int longestStreak = 0; // 用来记录“全球最高纪录”（最长连续长度）

        // 2. 遍历盒子里的每一个数字
        for(int num : num_set){
            
            // --- 核心逻辑：找“领头羊” ---
            // 如果盒子里面没有 (num - 1)，说明当前这个 num 就是一段连续数字的“起点”。
            // 比如有 1, 2, 3。看 2 的时候，发现 1 在里面，2 就不是起点，跳过。
            // 看到 1 的时候，发现 0 不在里面，OK，1 就是起点！
            if (!num_set.contains(num - 1)){
                
                int currentNum = num;      // 当前正在数的数
                int currentStreak = 1;     // 当前这一段的长度，刚开始肯定是 1

                // 3. 开始“接龙”
                // 问盒子：有没有比我大 1 的数？有的话，长度加 1，继续往后找。
                while (num_set.contains(currentNum + 1)){
                    currentNum += 1;       // 数字变大 1
                    currentStreak += 1;    // 长度增加 1
                }

                // 4. 更新最高纪录
                // Math.max 的意思是：取“原纪录”和“新纪录”里较大的那个。
                longestStreak = Math.max(longestStreak, currentStreak);
            }
        }

        // 5. 返回最终拿到的最高纪录
        return longestStreak;
    }
}

```

---

### 为什么要加 `if (!num_set.contains(num - 1))`？

这是这段代码效率极高的秘密。

如果没有这一行，假设数组是 `[1, 2, 3, 4, 5]`：

* 你会从 1 开始数，数到 5（走 5 步）。
* 然后你又从 2 开始数，数到 5（走 4 步）。
* 然后又从 3 开始数...
这样太慢了！

**有了这一行后：**

* 看到 2，发现有 1，不数了。
* 看到 3，发现有 2，不数了。
* **只有**看到 1 的时候，发现没有 0，才会一口气数到 5。
这样保证了数组里的每个数字最多只会被划过 **2 次**，速度变成了 $O(n)$。

### 知识点小本本

| 语法/方法 | 作用 |
| --- | --- |
| **`HashSet`** | 一个不重复的容器，查找（`contains`）速度极快。 |
| **`num_set.contains(x)`** | 检查数字 $x$ 是不是在容器里。 |
| **`Math.max(a, b)`** | 比较 $a$ 和 $b$，谁大留谁。 |
| **`while` 循环** | 只要条件满足（还能接龙），就一直跑下去。 |

### 总结

这段代码就像是在操场上找最长的一排排队学生：

1. 先让所有人站进集合区（`Set`）。
2. 只找那些**左边没人的同学**（`num - 1` 不存在），认定他们是排头。
3. 从排头开始往后数有多少人（`while` 接龙）。
4. 记下人数最多的那一排。
