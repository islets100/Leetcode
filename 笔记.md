这一段代码解决的是 **“字母异位词分组” (Group Anagrams)** 问题。

**什么是字母异位词？**
比如 `"eat"`、`"tea"` 和 `"ate"`，它们包含的字母完全一样（都是 a, e, t），只是顺序不同。这段代码的目标就是把这些“成分相同”的单词分到一组里。

---

### 核心思路：寻找“通用的钥匙”

要分组，就得有个标准。这段代码最聪明的地方在于：**把单词按字母排序。**

* `"eat"` 排序后是 `"aet"`
* `"tea"` 排序后也是 `"aet"`
* `"ate"` 排序后还是 `"aet"`

这样，`"aet"` 就成了这一组单词共同的 **Key（钥匙/标签）**。

---

### 逐行代码拆解

#### 1. 定义容器

```java
Map<String, List<String>> map = new HashMap<String, List<String>>();

```

* **`Map`**: 想象成一个有很多抽屉的柜子。
* **`String` (Key)**: 抽屉上的“标签”（即排序后的字符串，如 `"aet"`）。
* **`List<String>` (Value)**: 抽屉里装的“名单”（即原始单词列表，如 `["eat", "tea"]`）。

#### 2. 遍历每一个单词

```java
for (String str : strs) {

```

这是一个“增强型 for 循环”，意思是：把字符串数组 `strs` 里的单词，一个接一个拿出来处理，当前拿到的叫 `str`。

#### 3. 制造“标签” (排序)

```java
char[] array = str.toCharArray(); // 把 "eat" 变成 ['e', 'a', 't']
Arrays.sort(array);               // 把数组排成 ['a', 'e', 't']
String key = new String(array);   // 把数组变回字符串 "aet"

```

这里是关键：不管原始单词长什么样，只要字母相同，算出来的 `key` 就一定相同。

#### 4. 存入抽屉 (核心逻辑)

```java
List<String> list = map.getOrDefault(key, new ArrayList<String>());

```

* **`getOrDefault(key, default)`**: 这是一个非常实用的方法。
* 它会问 Map：“柜子里有 `key`（比如 `aet`）这个抽屉吗？”
* **如果有**：把抽屉里的名单拿出来（赋值给 `list`）。
* **如果没有**：现场新建一个空的名单（`new ArrayList`）。



```java
list.add(str);     // 把当前的原始单词（比如 "eat"）塞进名单里
map.put(key, list); // 把更新后的名单放回抽屉里

```

#### 5. 返回结果

```java
return new ArrayList<List<String>>(map.values());

```

* **`map.values()`**: 把柜子里所有抽屉里的“名单”全部取出来。
* **`new ArrayList(...)`**: 把这些名单打包成一个大列表返回。

### 常用成员方法总结表

如果你对 Java 基础不太熟悉，看这张表就能明白代码里那些“点（`.`）”后面跟着的函数在干嘛：

| 方法 | 所属类 | 作用 (白话解释) |
| --- | --- | --- |
| **`toCharArray()`** | `String` | 把一串话拆成一个个字符，方便排序。 |
| **`Arrays.sort()`** | `Arrays` | 自动把数组按字母表顺序 A-Z 排好。 |
| **`getOrDefault()`** | `Map` | 有就拿出来，没有就给个默认的（防止报错）。 |
| **`put(k, v)`** | `Map` | 往柜子里存东西，指定标签 `k` 和内容 `v`。 |
| **`add()`** | `List` | 往列表（名单）最后加一个新名字。 |
| **`values()`** | `Map` | 只要柜子里的内容，不要外面的标签。 |

---

### 总结

这段代码就像是一个邮局分拣员：

1. 拿到一个信封（单词）。
2. 把信封上的字母重新排个序（做成标签）。
3. 看看有没有这个标签的格堆。
4. 有就扔进去，没有就新开一个格堆。
5. 最后把所有的格堆一起交上去。
